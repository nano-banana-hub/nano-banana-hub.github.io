Agent Access

If an agent needs this hub, start with the stable files.

  1. Read llms.txt for curated entry points and ecosystem links.
  2. Use catalog.json for structured discovery, install commands, and source URLs.
  3. Open agent-catalog.md for a markdown digest of the current catalog.
  4. Install with bananahub add owner/repo/template-id.

Suggested install pattern

bananahub add nano-banana-hub/nanobanana/cute-sticker

BananaHub is aligned with Nano Banana's progressive-disclosure model: frontmatter stays lightweight for matching, while the full template body is only read when a template is activated.

Why BananaHub Exists

A template hub built for progressive loading, not template dumping.

The goal is not to flood Nano Banana Skill with every prompt variation. The goal is to keep the core skill lean, let templates live in GitHub, and expose clear discovery paths for humans, agents, and template authors.

01

Install only the capability you need

Templates are hot-pluggable. Users do not need to ship every domain recipe up front, and Nano Banana can stay focused until a real request calls for more.

02

Use tiered discovery instead of one giant prompt file

Lightweight frontmatter powers listing and matching. Richer body content remains in the source template and is pulled only when activation happens.

03

Give agents a first-class interface, not a hidden afterthought

The homepage is readable, but machine-readable files are linked directly so an agent can search, compare, and install templates without scraping visual UI.
